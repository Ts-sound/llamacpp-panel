# llamacpp-panel Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build a complete lightweight GUI management tool for llama.cpp with process control, parameter configuration, memory monitoring, auto-restart, and SSH port mapping.

**Architecture:** Four-layer architecture — Models (data-only @dataclass) → Services (business logic, callbacks) → UI (Tkinter, calls services only) → Utils (file ops, cross-platform). Config in `config.py`. Strict separation: no UI→Model direct access, no business logic in models.

**Tech Stack:** Python 3.9+, Tkinter (builtin), psutil, subprocess, threading, json
