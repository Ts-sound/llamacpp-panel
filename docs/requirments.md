
# 需求文档
## 1. 项目概述
### 1.1 项目定位
面向 llama.cpp 终端用户的**轻量化GUI管理工具**，替代纯命令行操作，实现 llama.cpp 服务的可视化管控，核心解决命令行参数繁琐、进程无监控、无自动恢复、远程映射配置复杂的问题。
### 1.2 技术栈
- 开发语言：Python 3.9+
- GUI框架：Python 内置 `Tkinter`（无额外安装，跨平台）
- 核心依赖：`psutil`（内存监控）、`subprocess`（进程调用）、`threading`（后台多线程）、`json`（配置持久化）
- 目标平台：Windows / Linux（跨平台兼容）

## 2. 核心功能需求
### 2.1 启动文件选择模块
1. 支持通过**文件对话框**可视化选择 llama.cpp 编译后的可执行文件（`server`）；
2. 自动记录历史选择的文件路径，支持快速切换；

### 2.2 启动参数配置模块
1. 可视化**添加/删除/编辑** llama.cpp 启动参数（如 `-m 模型路径`、`-c 上下文长度`、`-ngl 显卡层` 等）；
2. 预设一些常用基本参数，参考：https://github.com/ggml-org/llama.cpp/blob/master/tools/server/README.md
3. 提供**常用参数预设模板**，一键加载；
4. 自动拼接参数为合法启动命令，实时预览最终命令；
5. 支持参数配置保存/加载，无需重复设置。

### 2.3 内存实时监控模块
1. 实时监控**系统物理内存使用率**（数值+进度条可视化）；
2. 可选监控GPU显存（适配NVIDIA/CPU模式）；
3. 支持内存阈值告警（颜色标红/弹窗提示）；
4. 监控数据轻量刷新（3秒/次），无性能损耗。

### 2.4 服务自动重启模块
1. 支持开关**自动重启功能**；
2. 重启触发条件：llama.cpp 进程**异常崩溃**、**内存超阈值**；
3. 可配置最大重启次数，防止无限循环；
4. 记录重启日志，展示重启原因、时间。

### 2.5 SSH端口映射模块
1. 一键执行标准反向映射命令：`ssh -R 0.0.0.0:8080:127.0.0.1:8080 root@172.18.122.71`；
2. 支持自定义：本地端口、远程端口、远程服务器IP、登录用户名；
3. 实时显示SSH连接状态（连接中/已连接/断开）；
4. 后台保持SSH连接，随 llama.cpp 服务同步启动/停止。

### 2.6 通用基础功能
1. 一键**启动/停止** llama.cpp 服务；
2. 实时输出 llama.cpp 运行日志、SSH日志、系统日志；
3. 配置文件自动保存（重启软件后保留上次设置）；
4. 界面简洁易用，无学习成本。

## 3. 非功能需求
1. **轻量无依赖**：仅需Python环境，GUI使用系统自带库，无需安装额外包；
2. **低资源占用**：后台内存占用＜100MB，不影响 llama.cpp 运行；
3. **稳定可靠**：多线程分离GUI与进程，界面不卡顿、不崩溃；
4. **跨平台兼容**：完美支持Windows 10/11、Linux桌面版；
5. **易用性**：所有功能可视化，无需命令行基础即可操作。

## 4. 界面设计规划
采用 Tkinter 分模块布局，简洁直观：
1. 总看板：内存监控仪表盘 + 一键启动/停止按钮 + 日志框
2. 参数配置：启动文件文件选择 + 参数配置面板（表格/输入框）；
2. SSH端口映射配置：SSH端口映射配置区；

## 5. 交付标准
1. Python脚本/轻量化打包文件，开箱运行；
2. 附带极简使用说明；
3. 所有核心功能无BUG，稳定运行。
